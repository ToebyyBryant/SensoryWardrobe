# Sensory Wardrobe

A mobile-first Flutter app that helps sensory-sensitive individuals and their caregivers make comfortable clothing decisions every day — informed by personal comfort history and real-time weather.

**Team:** Bruce Schulz, Nova Denton-Parry, Zach Kahler, Jeremy Kirkpatrick  
**Course:** CIS248 Advanced App Development — Summer 2026

---

## Overview

Sensory Wardrobe is designed for individuals with sensory processing differences (autism, SPD) and their caregivers. Users build a wardrobe catalog tagged with sensory attributes (texture, pressure, warmth), log daily outfits, rate post-wear comfort, and receive weather-appropriate outfit suggestions that improve over time based on their comfort history.

```

│
├── core/

│   ├── database/              # SQLite DatabaseHelper (DS1–DS5, schema v1→v2 migration)
│   ├── error/                 # Typed AppException hierarchy
│   ├── network/               # ApiClient (generic HTTP wrapper)
│   ├── router/                # GoRouter setup + auth guard redirect
│   └── theme/                 # AppTheme (light + dark) + AppColors
│
├── features/
│   ├── auth/                  # P1.0 — Login, register, auth state providers
│   ├── wardrobe/              # P2.0 — Clothing catalog CRUD + filter + image picker
│   ├── weather/               # P3.0 — Fetch + cache weather snapshots
│   ├── outfit_log/            # P4.0 — Daily outfit logging + comfort ratings
│   ├── suggestions/           # P5.0 — Smart suggestion engine + providers
│   ├── history/               # P6.0 — Outfit log list + comfort trend charts
│   ├── notifications/         # P7.0 — Local notification scheduling (timezone-aware)
│   ├── backup/                # P8.0 — Firebase cloud backup/restore (encrypted)
│   ├── admin/                 # P9.0 — Admin panel (user management, system config)
│   ├── onboarding/            # First-run onboarding flow
│   ├── profile/               # Profile management + multi-profile switching

Each feature follows the internal pattern:
```

├── data/

│   ├── repositories/    # SQLite + API access
│   └── services/        # Business logic (suggestion engine, backup service)
└── presentation/
    ├── providers/       # Riverpod StateNotifier / StreamProvider / FutureProvider

## Data Flow

The app follows the data flow defined in the Level 0 DFD:

1. **User** provides clothing items, outfit selections, comfort ratings, and preferences
2. **Weather API** supplies real-time temperature, humidity, and conditions
3. **Suggestion Engine** combines wardrobe catalog + weather context + historical comfort scores to rank outfit recommendations
4. **Notification Service** reminds users to log outfits (morning) and rate comfort (evening)
5. **Backup Service** encrypts and syncs all user data to Firebase cloud storage

### External Entities
- Sensory-Sensitive User (primary)
- Caregiver (manages dependent profiles)
- Admin (system configuration)
- OpenWeatherMap API
- Firebase Cloud (auth, storage, Firestore)
- Push Notification Service

### Clothing Categories
Top, Bottom, Underwear, Socks, Shoes, Jacket/Outerwear, Dress/Romper, Sleepwear, Accessories

### Sensory Tags

## Getting Started

### Prerequisites
- Flutter SDK ≥ 3.0
- Android Studio or Xcode (for device/emulator)
- A Firebase project with Auth, Firestore, and Storage enabled
- An OpenWeatherMap API key (free tier works fine)

### Setup

1. **Clone and install dependencies**
   ```bash
   cd sensory_wardrobe
   flutter pub get
   ```

2. **Configure Firebase**
   ```bash
   dart pub global activate flutterfire_cli
   flutterfire configure --project=YOUR_FIREBASE_PROJECT_ID
   ```
   This generates `lib/firebase_options.dart` and places `google-services.json` / `GoogleService-Info.plist`.

3. **Set your OpenWeatherMap API key**  
   The key is stored in Flutter Secure Storage (never hard-coded). On first launch go to **Settings → Weather API Key** and paste it in.

4. **Run code generation** (for Riverpod providers and JSON serialization)
   ```bash
   dart run build_runner build --delete-conflicting-outputs
   ```

5. **Run the app**
   ```bash
   flutter run
   ```

## Suggestion Algorithm

The suggestion engine (P5.0) works as follows:

1. Map current temperature to a target warmth level (0°C→5, 10°C→4, 18°C→3, 25°C→2, 30°C+→1)
2. Filter wardrobe items to those within ±1 warmth level of target
3. Score each item by its historical average comfort rating (unknown items default to 3.0)
4. Sort candidates by descending comfort score
5. Return the ranked list as suggestions

When weather data is unavailable, the engine falls back to comfort-score-only sorting.
